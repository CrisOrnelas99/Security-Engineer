# What I Learned — Information Security Frameworks & Governance Documents

## Information Security Framework

- A structured set of documents defining how security is:
  - Implemented
  - Managed
  - Enforced within an organisation.

### Core Components:

- **Policies** → High-level formal statements of goals and principles.
- **Standards** → Mandatory technical or procedural requirements.
- **Guidelines** → Recommended (non-mandatory) best practices.
- **Procedures** → Step-by-step instructions to complete tasks.
- **Baselines** → Minimum required security configurations or controls.

---

## How Governance Documents Are Developed

1. **Identify Scope & Purpose**
   - Define what the document covers and why it’s needed.

2. **Research & Review**
   - Consider laws, regulations, industry standards, and existing internal documents.

3. **Draft the Document**
   - Clear, concise, actionable.
   - Aligned with business objectives.

4. **Review & Approval**
   - Involve stakeholders (legal, compliance, SMEs, management).
   - Final approval by authorised leadership.

5. **Implementation & Communication**
   - Communicate responsibilities.
   - Provide training and awareness.

6. **Review & Update**
   - Periodic updates.
   - Adjust based on regulatory or threat changes.

---

## Example 1 — Password Policy

- Define:
  - Minimum length
  - Complexity
  - Expiration rules
- Usage rules:
  - No password sharing
  - No default passwords
  - Unique passwords per system
- Secure storage & transmission:
  - Encryption
  - Secure connections (e.g., TLS)
- Reset/change process defined.
- Communicate policy.
- Monitor compliance and update as needed.

---

## Example 2 — Incident Response Procedure

- Define incident types:
  - Unauthorised access
  - Malware
  - Data breaches
- Define roles & responsibilities.
- Create step-by-step response process:
  - Containment
  - Investigation
  - Mitigation
  - Recovery
- Report and document incidents.
- Communicate procedure.
- Regularly review and improve.

---

## Using External Standards & Frameworks

- Organisations often adopt existing standards instead of building from scratch.
- Choice depends on:
  - Industry (e.g., finance, healthcare)
  - Regulatory requirements
  - Geographic region
  - Scope and objectives
  - Available resources

---

## Key Takeaway

- Security frameworks provide structured governance.
- Policies define *what* must be done.
- Standards define *mandatory requirements*.
- Guidelines suggest *best practices*.
- Procedures define *how to execute tasks*.
- Baselines define *minimum acceptable security level*.
