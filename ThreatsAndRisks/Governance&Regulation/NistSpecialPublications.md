# What I Learned — NIST 800-53 and NIST 800-63B

## NIST SP 800-53

- Full title: "Security and Privacy Controls for Information Systems and Organizations".
- Developed by NIST (US).
- Provides a comprehensive catalogue of security and privacy controls.
- Purpose:
  - Protect the CIA triad (Confidentiality, Integrity, Availability).
  - Help organisations comply with laws, regulations, and policies.
  - Strengthen information system security and privacy.

### Key Characteristics

- Covers protection against:
  - Intentional attacks
  - Human error
  - Natural disasters
  - Infrastructure failures
  - Foreign intelligence threats
  - Privacy risks
- Revision 5 organises controls into 20 control families.
- Controls are mapped to different security and privacy objectives.

---

## Program Management Control (Important Family)

- One of the key control families in NIST 800-53.
- Requires organisation-wide security and privacy programs.
- Focus:
  - Establish
  - Implement
  - Monitor
  - Maintain security governance

- Ensures systems that process, store, or transmit data are protected.
- Requires structured oversight, accountability, and continuous evaluation.

---

## NIST 800-53 Compliance Best Practices

1. Identify and catalogue:
   - Data assets
   - Information systems
   - Threats and vulnerabilities
   - Data flows and dependencies

2. Map control families to:
   - Identified assets
   - Identified risks

3. Establish governance structure:
   - Define roles and responsibilities
   - Define implementation procedures

4. Monitor and evaluate:
   - Conduct regular audits
   - Assess control effectiveness
   - Improve continuously

5. Implement monitoring systems:
   - Detect security issues
   - Respond to incidents
   - Maintain compliance posture

## NIST SP 800-63B

- Focuses on digital identity and authentication.
- Provides guidelines for:
  - Identity verification
  - Authentication mechanisms
  - Credential management

### Key Concepts

- Defines different identity assurance levels (low to high).
- Recommends secure use of:
  - Passwords
  - Biometrics
  - Tokens
- Emphasises:
  - Secure storage of credentials
  - Strong authentication practices
  - Risk-based identity management

---

## Key Takeaway

- NIST 800-53 = Comprehensive security and privacy control framework.
- NIST 800-63B = Focused on digital identity and authentication security.
- Both frameworks help organisations strengthen security governance and compliance.
